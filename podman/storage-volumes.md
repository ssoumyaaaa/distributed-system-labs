# Storage & Volumes in Podman

Storage in Podman is used to persist data generated by containers. By default, container data is ephemeral, meaning it is lost when the container is removed.

---

## 💾 Why Volumes?

Without volumes:
- Data is lost when container stops or is deleted

With volumes:
- Data persists outside container lifecycle
- Can be shared between containers

---

## 📦 Types of Storage

| Type | Description |
|------|------------|
| Volume | Managed by Podman |
| Bind Mount | Host directory mapped to container |
| tmpfs | Temporary in-memory storage |

---

## 📌 Create a Volume

```bash
podman volume create myvol
```

---

## 📂 List Volumes

```bash
podman volume ls
```

---

## 🔍 Inspect Volume

```bash
podman volume inspect myvol
```

---

## 🚀 Use Volume in Container

```bash
podman run -d -v myvol:/data alpine sleep 1000
```

---

## 🧭 Bind Mount (Host Path)

```bash
podman run -d -v /home/user/data:/data alpine sleep 1000
```

- Left side → Host path
- Right side → Container path

---

## 🧪 tmpfs (Temporary Storage)

```bash
podman run -d --tmpfs /data alpine sleep 1000
```

Data is stored in memory only.

---

## 🧹 Remove Volume

```bash
podman volume rm myvol
```

Remove unused volumes:

```bash
podman volume prune
```

---

## 📌 Example: Persistent Nginx Data

```bash
podman volume create webdata
podman run -d -p 8080:80 -v webdata:/usr/share/nginx/html nginx
```

---

## 📊 Check Mounted Volumes

```bash
podman inspect container_id | grep Mounts -A 10
```

---

## 🧠 Key Concepts

- Containers are ephemeral by default
- Volumes provide persistence
- Bind mounts connect host and container filesystem
- tmpfs is memory-based storage

---

## 🛠️ Practice

```bash
podman volume create testvol
podman run -it -v testvol:/data alpine sh
```

Inside container:

```bash
echo "hello" > /data/file.txt
```

---

## 🎯 Why This Matters

- Required for databases (MySQL, MongoDB, etc.)
- Used in Kubernetes persistent volumes
- Essential for stateful applications
- Critical for production systems
