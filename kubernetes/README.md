# Kubernetes

A practical guide to learning and implementing **Kubernetes** using hands-on labs.

This documentation is based on a real lab environment created using **Kind**, **Podman Desktop**, and **Rocky Linux Virtual Machines**.

---

## Lab Environment

| Component | Details |
|----------|---------|
| Kubernetes | Kind Cluster |
| Runtime | Podman Desktop |
| Operating System | Rocky Linux 9 |
| CLI | kubectl |
| Container Images | Docker Hub |
| Editor | Visual Studio Code |
| Terminal | Bash |

## Learning Outcomes

After completing this documentation, you will be able to:

- Understand Kubernetes architecture
- Create and manage Kubernetes clusters
- Deploy containerized applications
- Scale applications using Deployments
- Expose applications using Services
- Configure applications using ConfigMaps and Secrets
- Manage persistent storage
- Deploy applications using Helm
- Monitor and troubleshoot Kubernetes resources
- Follow Kubernetes best practices

---

## Prerequisites

- Basic Linux commands
- Basic Docker knowledge
- Docker or Podman installed
- kubectl installed
- Kind installed

> **Note:** It is recommended to complete the **Docker** module before starting Kubernetes.
